# 户外运动管理平台项目开发计划

## 1. 项目里程碑 (Milestones)

| 阶段 | 周期 | 核心目标 | 交付物 |
| :--- | :--- | :--- | :--- |

## 2. 第一阶段 (M1-M2) 详细 WBS (工作分解结构)

### 2.1 基础设施与环境 (Infrastructure)
- [ ] **INF-01**: 项目仓库初始化 (Git)
- [ ] **INF-02**: 后端 NestJS 框架搭建 (含 Prisma, Swagger, Docker配置)
- [ ] **INF-03**: 前端 Taro 框架搭建 (含 NutUI, Redux/Zustand, 自定义TabBar)
- [ ] **INF-04**: 数据库环境准备 (PostgreSQL + PostGIS)

### 2.2 数据库设计 (Database)
- [ ] **DB-01**: 设计用户表 (User) 与 认证表 (Auth)
- [ ] **DB-02**: 设计活动表 (Activity) 与 报名表 (Enrollment)
- [ ] **DB-03**: 设计线路表 (Route) 基础结构

### 2.3 后端开发 (Backend)
- [ ] **BE-01**: 实现微信登录接口 (AuthService)
- [ ] **BE-02**: 实现文件上传接口 (OSS集成，用于图片上传)
- [ ] **BE-03**: 活动管理 API (CRUD)
    - [ ] 发布活动
    - [ ] 获取活动列表 (分页、筛选)
    - [ ] 获取活动详情
- [ ] **BE-04**: 报名 API
    - [ ] 创建报名记录
    - [ ] 取消报名
    - [ ] 获取我的报名列表

### 2.4 小程序前端开发 (Frontend)
- [ ] **FE-01**: 全局样式与主题配置 (NutUI Theme)
- [ ] **FE-02**: 封装通用网络请求 (Request Interceptor)
- [ ] **FE-03**: 底部导航栏 (TabBar) 实现
- [ ] **FE-04**: **首页 (Home)**
    - [ ] Banner 轮播图
    - [ ] 快捷入口 (徒步、登山、露营)
    - [ ] 推荐活动列表 (卡片式布局)
- [ ] **FE-05**: **活动详情页 (Activity Detail)**
    - [ ] 顶部大图
    - [ ] 关键信息 (时间、地点、难度、价格)
    - [ ] 详情富文本/长图展示
    - [ ] 底部悬浮报名栏
- [ ] **FE-06**: **个人中心 (Profile)**
    - [ ] 用户信息展示 (头像、昵称)
    - [ ] 菜单列表 (我的活动、我的线路、联系客服)
- [ ] **FE-07**: **登录流程**
    - [ ] 微信一键登录按钮
    - [ ] 用户信息补全页

### 2.5 验收标准 (Acceptance Criteria)
1.  用户可以通过微信授权登录。
2.  首页能展示模拟的活动列表。
3.  点击活动卡片能进入详情页查看完整信息。
4.  点击“立即报名”能模拟报名成功并跳转到订单页。
5.  个人中心能看到已报名的活动。
